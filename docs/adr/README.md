# Архитектурные решения (ADR)

## Зачем это

**ADR (Architecture Decision Record)** фиксирует значимое решение по архитектуре: контекст, выбранный вариант, отвергнутые альтернативы и последствия. Документ остаётся в репозитории, чтобы новые участники и будущие мы понимали *почему* система устроена именно так.

## Принципы проекта

- Одно решение — один файл `adr-NNN-краткое-имя.md`, где `NNN` — порядковый номер с ведущими нулями.
- Статус указывается в шапке: черновик, предложено, принято, устарело (с ссылкой на замену, если есть).
- Формат: контекст → решение → (по возможности) альтернативы и последствия. Без кода и низкоуровневых деталей реализации — их оставляем коду и специализированным документам.
- ADR **не переписывают задним числом**: при смене решения добавляется новый ADR, который отменяет или уточняет предыдущий.
- Сводный список ниже — для навигации; источник правды по каждому пункту — отдельный файл.
