# Backend Tasklist

## Обзор

**Backend** — ядро системы: доменная логика, сессии и прогресс, вызовы LLM, доступ к данным. Через REST API к нему подключаются **все клиенты** (Telegram-бот, в будущем веб) и концентрируются **интеграции** с внешними системами (OpenRouter и др. по [integrations.md](../integrations.md)). Этот tasklist ведёт **итерации 2, 3 и 5** дорожной карты ([plan.md](../plan.md)) в виде последовательных волн внутри области backend.

## Легенда статусов

- 📋 **Planned** — запланирован
- 🚧 **In Progress** — в работе
- ✅ **Done** — завершён

## Качество и инженерные практики

- **Тесты:** модульные/интеграционные тесты для критичных сервисов и API (минимум — happy-path и ошибки валидации); по мере появления БД — тесты репозиториев или контрактные с миграциями.
- **Линтеры и стиль:** `ruff` ([vision.md](../vision.md)); единый стиль импортов и типов в новом коде backend.
- **Наблюдаемость:** структурированное логирование (`LOG_LEVEL`), идентификаторы запроса/пользователя в логах где уместно; без утечки секретов.
- **Изменения контрактов:** обратная совместимость или явная версия API; крупные ломающие изменения — через ADR и обновление OpenAPI до деплоя клиентов.
