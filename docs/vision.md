# Техническое видение: AI-репетитор (Telegram-бот)

## Технологии

### Язык и окружение
- Python 3.12+
- Виртуальное окружение: `venv`
- Управление зависимостями: `uv`
- Файл зависимостей: `requirements.txt`

### Зависимости

| Пакет | Назначение |
|---|---|
| `aiogram` | Telegram Bot API, polling |
| `openai` | OpenAI-совместимый клиент для OpenRouter |
| `pydantic-settings` | Загрузка и валидация конфигурации |
| `python-dotenv` | Поддержка `.env` |

### Make-команды

| Команда | Действие |
|---|---|
| `make install` | Создать venv и установить зависимости через uv |
| `make run` | Запустить бота |
| `make lint` | Проверка кода (ruff check) |
| `make format` | Форматирование (ruff format) |

## Принципы разработки

- **KISS** — минимальная сложность, без оверинжиниринга.
- **Один класс — один файл** — каждый нетривиальный класс живёт в отдельном модуле.
- **Явный конфиг** — все настройки через `.env`, хардкод запрещён.
- **Stateless LLM-вызовы** — история диалога хранится в памяти на время сессии; LLM-клиент ничего не помнит сам по себе.
- **Без БД в MVP** — состояние только в памяти процесса; база данных добавляется позже по необходимости.

## Архитектура

Бот работает в режиме **polling**. Входящее сообщение обрабатывается обработчиком aiogram, который обращается к `TutorSession` за контекстом и вызывает `LLMClient` для генерации ответа.

```
Telegram → aiogram (polling) → Handler → TutorSession → LLMClient → OpenRouter → ответ пользователю
```

**Ключевые компоненты:**
- `main.py` — инициализирует бота, диспетчер, регистрирует роутеры, запускает polling.
- `config.py` — класс `Settings` (pydantic-settings), читает `.env`.
- `llm/client.py` — класс `LLMClient`: отправляет запросы, возвращает строку-ответ.
- `tutor/session.py` — класс `TutorSession`: хранит историю, уровень, пройденные темы.
- `bot/handlers/` — функции-обработчики; каждый файл отвечает за один смысловой блок (старт, объяснение, тест, прогресс).

## Модель данных / Состояние

В MVP состояние хранится **в памяти** — глобальный словарь `user_id → TutorSession`.

```python
@dataclass
class TutorSession:
    user_id: int
    level: str           # "school" | "adult" | "unknown"
    topic: str           # текущая тема
    history: list[dict]  # messages[] для LLM (role/content)
    progress: list[str]  # список пройденных тем за сессию
```

При перезапуске бота состояние сбрасывается — это приемлемо для MVP.

## Работа с LLM

- **Провайдер**: [OpenRouter](https://openrouter.ai) — единая точка доступа к множеству моделей.
- **Клиент**: `openai.OpenAI` с кастомными `base_url` и `api_key` (OpenAI-совместимый интерфейс).
- **Модель по умолчанию**: задаётся в `.env` (например, `openai/gpt-4o-mini`).
- **Системный промпт**: задаёт роль репетитора и стиль — подстраивает тон под уровень ученика.

```python
class LLMClient:
    def __init__(self, api_key: str, base_url: str, model: str): ...
    def chat(self, messages: list[dict]) -> str: ...
```

Метод `chat` принимает готовый массив `messages[]` и возвращает текст ответа. Формирование истории — задача `TutorSession`.

## Сценарии работы пользователя

1. **Объяснение темы** — ученик пишет «объясни дроби» → бот объясняет на нужном уровне.
2. **Проверка знаний** — «дай тест по теореме Пифагора» → бот генерирует задания и проверяет ответы.
3. **Разбор домашнего задания** — ученик присылает своё решение → бот находит ошибки и объясняет.
4. **Просмотр прогресса** — «что я уже прошёл?» → бот перечисляет темы из текущей сессии.

## Логирование

- Стандартный модуль `logging`, настройка в `main.py`.
- Уровень задаётся через `LOG_LEVEL` в `.env` (по умолчанию `INFO`).
- Формат: `%(asctime)s [%(levelname)s] %(name)s: %(message)s`.
- В MVP — вывод в stdout.
